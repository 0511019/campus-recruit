# 高校招聘信息聚合平台 — 产品需求文档 (PRD)

> **版本**：v1.0  
> **日期**：2026-07-20  
> **作者**：企业HR需求驱动  

---

## 一、产品定位

### 1.1 产品名称

**校招瞭望台**（暂定名）

### 1.2 一句话定位

为企业HR提供**一站式高校招聘活动信息聚合与订阅**的平台——告别逐个高校就业网刷信息的低效模式。

### 1.3 目标用户

| 用户角色 | 描述 |
|---------|------|
| **企业HR** | 负责校园招聘的HR、招聘专员、校招负责人，需要及时获取目标高校/区域的招聘活动信息并完成企业端报名 |

### 1.4 核心痛点

1. **信息分散**：广东省（乃至全国）高校100+所，每所高校就业网独立发布招聘会信息，HR需要逐个人工浏览
2. **时效性差**：错过报名截止时间导致无法参展，或发现时展位已满
3. **筛选困难**：无法按区域、届数、活动类型快速筛选，"2026届毕业生暨2027届实习生"这种跨届活动容易遗漏
4. **缺乏提醒**：没有自动通知机制，依赖人工定期巡检

### 1.5 与竞品的差异

| 对比维度 | 各高校就业网 | 智联/前程无忧 | **校招瞭望台** |
|---------|-------------|--------------|---------------|
| 信息范围 | 单校 | 已入驻企业 | 跨校聚合 |
| 面向对象 | 学生为主 | 学生+企业 | **企业HR为主** |
| 报名信息 | ✅ | ❌ 仅展示 | ✅ **重点呈现** |
| 订阅通知 | ❌ | 部分 | ✅ **核心功能** |

## 二、产品目标

### 2.1 北极星指标

**企业HR从"发现活动"到"完成报名"的时间缩短80%**（从当前人工巡检的30分钟/天降至5分钟/天）

### 2.2 阶段目标

| 阶段 | 目标 | 衡量指标 |
|------|------|----------|
| **MVP（1-2周）** | 广东省高校 + 2027届活动聚合与搜索 | 覆盖30+广东高校，聚合10+有效活动 |
| **V1.0（1个月）** | 扩展到华南四省 + 多届别，接入订阅通知 | 覆盖100+高校，月活HR 50+ |
| **V2.0（3个月）** | 全国高校覆盖 + 智能推荐 + 报名DDL预警 | 覆盖全国重点高校500+ |

   ┌─ 活动类型 ────────────┐

#### 触发通知的时机

| 触发条件 | 通知内容 |
|---------|---------|
| 新活动上线（匹配订阅条件） | "【新活动】华南理工大学2027届秋季双选会-企业报名已开启" |
| 报名即将截止（距截止 < 3天） | "⚠️【即将截止】华农暑期空中双选会企业报名还剩2天" |
| 活动状态变更（预告→报名中） | "【报名已开放】华南理工大学2027届秋季双选会" |

### 3.3 辅助功能（P1）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **收藏夹** | HR可收藏感兴趣的活动，集中查看 | P1 |
| **历史浏览记录** | 记录最近查看的活动，方便回溯 | P1 |
| **活动日历视图** | 以日历形式展示活动排期 | P1 |
| **一键复制报名信息** | 复制活动摘要+报名链接，方便分享给同事 | P2 |
| **数据导出** | 筛选结果导出为 Excel/CSV | P2 |
| **报名DDL倒计时** | 结果列表中醒目标注距截止还有几天 | P2 |

### 3.4 数据管理（P0 后台）

| 功能 | 描述 |
|------|------|
| **数据采集** | 自动采集（RPA/爬虫）+ 人工录入 + 用户贡献 |
| **数据核验** | 标记信息是否已验证，避免过期/错误信息误导 |
| **活动状态自动更新** | 根据时间自动切换：预告→报名中→进行中→已结束 |
| **重复检测** | 同一活动被多校转发时自动去重合并 |

## 六、异常情况与边界处理

### 6.1 数据层面

| 异常场景 | 处理策略 |
|---------|---------|
| **同一活动被多校转发** | 系统通过标题相似度 + 报名URL匹配自动去重，合并到一个活动下展示多个源链接 |
| **活动信息不完整**（如缺少企业报名方式） | 标记为 `⚠️ 信息待补全`，鼓励用户"补充信息"（UGC贡献），同时系统尝试自动抓取 |
| **源链接失效/404** | 定期巡检（每24h），失效链接标记 `🔗 链接已失效`，自动隐藏或降权 |
| **活动跨届**（如"2026届暨2027届"） | 存储为数组 `["2026届", "2027届"]`，搜索任一届数均可命中 |
| **活动形式混合**（线上+线下皆有） | `format: "hybrid"`，同时展示平台和地点 |
| **报名链接非标准格式**（邮箱报名/微信报名） | 支持 `报名方式: "邮箱投递"` 并在卡片中醒目标注 |

### 6.2 用户层面

| 异常场景 | 处理策略 |
|---------|---------|
| **筛选条件无结果** | 提示"暂未找到匹配活动，建议放宽筛选条件或设置订阅提醒"，附带快速订阅入口 |
| **订阅触发过于频繁** | 同类型通知合并（如"今日新增3场广东双选会"），支持设置通知频率（实时/每日摘要/每周摘要） |
| **邮件通知被拒收** | 定期检测退信率，提醒用户检查垃圾邮件箱或切换通知方式 |
| **报名截止时间已过，用户仍尝试访问** | 活动卡片标记为"已结束"，报名链接灰色处理但仍可查看历史信息 |

### 6.3 系统层面

| 异常场景 | 处理策略 |
|---------|---------|
| **数据采集失败**（源站屏蔽/改版） | 记录采集日志，人工介入修复，告警通知管理员 |
| **高并发/流量突增** | 首页+搜索结果缓存（TTL: 30min），详情页实时渲染 |
| **非法数据/垃圾信息**（如用户贡献功能遭滥用） | 用户贡献需后台审核后上架；异常行为风控拦截 |

## 七、非功能需求

| 维度 | 要求 |
|------|------|
| **性能** | 首屏加载 < 2s，搜索结果 < 1s |
| **可用性** | 支持 PC Web + 移动端响应式（HR在外勤时也能查看） |
| **数据更新频率** | 核心高校就业网数据每 6h 巡检一次 |
| **通知时效** | 新活动上线后 30min 内推送通知 |
| **多语言** | V1.0 仅支持中文 |
| **合规** | 所有数据来源于公开的高校就业网，注明源链接，遵循 robots.txt |
